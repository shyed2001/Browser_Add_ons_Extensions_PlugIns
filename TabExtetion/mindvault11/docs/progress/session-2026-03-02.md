## v4.5.0 тАФ Machine Sync (Bidirectional Force Sync)

### Problem
- No way to force-push all browser IndexedDB sessions to companion SQLite in one shot
- Companion had no mechanism to request browsers to sync
- Custom libraries were silently failing to sync (404 on session push тАФ library didn't exist in SQLite)

### Solution
Two buttons + a polling mechanism:
- **Extension Dashboard `[тШБ Machine Sync]`** тАФ force-pushes ALL this browser's sessions (ignores `syncedToCompanion` flag), creates missing libraries in companion first
- **Companion All Tabs `[тШБ Machine Sync]`** тАФ posts a sync request to daemon; extension SW polls every 30s, detects it, runs forceAllSync(), notifies done; companion auto-refreshes All Tabs

| # | Task | File(s) |
|---|------|---------|
| 1 | `POST /sync`, `GET /sync/pending`, `POST /sync/done` handlers + in-memory sync state (`sync.Mutex`) | `handlers.go` |
| 2 | Register 3 new routes | `server.go` |
| 3 | `forceAllSync()`, `checkSyncPending()`, `notifySyncDone()` functions | `companion-client.ts` |
| 4 | 30s `setInterval` polling in SW тАФ detects pending sync, runs forceAllSync, notifies done | `background/index.ts` |
| 5 | `[тШБ Machine Sync]` button in Extension Dashboard header | `dashboard.html`, `dashboard/index.ts`, `dashboard.css` |
| 6 | `[тШБ Machine Sync]` button + sync notice bar in Companion All Tabs toolbar | `companion/internal/api/ui/index.html`, `app.js`, `style.css` |
| 7 | **Bug fix**: custom libraries not syncing to companion тАФ added `libraryExistsInCompanion()` check + `POST /libraries` before pushing sessions in both `syncAllUnpushedSessions` and `forceAllSync` | `companion-client.ts` |

**Sync state:** package-level Go vars (`syncPending`, `requestedAt`, `syncDoneAt`) guarded by `sync.Mutex` тАФ ephemeral, no DB migration

## v4.5.1 тАФ Companion All Tabs Performance

### Problem
All Tabs panel was slow because:
1. `renderMasterTabsTable()` recreated ALL 364+ rows ├Ч 11 columns = 4 000+ DOM nodes on every panel switch, search keystroke, or sync completion
2. `loadMasterTabs()` hit the server on every panel switch (no caching)
3. No search debounce тАФ full re-render on every character typed

### Solution

| # | Fix | Before | After |
|---|-----|--------|-------|
| 1 | Pagination | All rows rendered | First 200, "тмЗ Load more (N remaining)" button |
| 2 | Data cache | Re-fetch every panel switch | 30s TTL; `loadMasterTabs(true)` to force refresh |
| 3 | Search debounce | Re-render per keystroke | 250ms delay |
| 4 | Sort page reset | No reset | `masterTabsPage = 1` on column sort click |
| 5 | Machine Sync refresh | Used stale cache | `loadMasterTabs(true)` after sync completes |
| 6 | Delete tab cache sync | Cache out of date | `masterTabsCache = allMasterTabs` after filter |

**New state vars:** `masterTabsCache`, `masterTabsCacheAt`, `MASTER_TABS_TTL=30000`, `PAGE_SIZE=200`, `masterTabsPage`, `_searchDebounce`  
**New CSS:** `.tt-load-more` (full-grid-width row, hover highlight)  
**Files:** `companion/internal/api/ui/app.js`, `companion/internal/api/ui/style.css`  
**No Go changes** тАФ purely frontend

## Architecture Decisions (confirmed this session)

| Decision | Choice | Reason |
|----------|--------|--------|
| Extension storage | **IndexedDB** тАФ keep forever | Browser sandbox, offline-first, no change needed |
| Companion storage | **SQLiteDB** local file | Central local store + offline fallback for cloud |
| DB path | `%APPDATA%\MindVault\db.sqlite` | Confirmed from code (not %LOCALAPPDATA%) |
| Backup dir | `%APPDATA%\MindVault\backups\` | Adjacent to db.sqlite |
| Cloud PWA storage | PostgreSQL (server) + wa-sqlite/OPFS (client) | Offline-capable PWA without IndexedDB limitations |
| Mobile MAUI | SQLiteDB local + cloud sync | Cross-platform offline-capable |
| Native Messaging | **DO NOT implement yet** | Already stubbed (`--native` flag + TODO(step-12) in main.go) тАФ Phase 5 only |

### Three-Tier Storage Design (Phase 5 target)
```
Extension (IndexedDB, per browser) тЖТ Machine Sync тЖТ Companion SQLite (local central)
                                                           тЖУ (Phase 5)
                                            Cloud PostgreSQL + wa-sqlite PWA cache
```
